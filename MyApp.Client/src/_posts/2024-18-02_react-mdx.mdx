### Modern React

Modern React takes advantage of advancements in JavaScript, TypeScript and the React library
itself to quickly develop highly interactive, modular applications.

Where creating a component is as simple as:

**[Hello.tsx](https://github.com/NetCoreTemplates/react-spa/blob/main/MyApp.Client/src/_posts/components/Hello.tsx)**

```tsx
type Props = { name:string }
export default ({ name }:Props) => <b>Hello, {name}!</b>
```
<div className="text-center text-2xl py-2">
    <Hello name="React" />
</div>

Or a simple reactive example:

**[Counter.tsx](https://github.com/NetCoreTemplates/react-spa/blob/main/MyApp.Client/src/_posts/components/Counter.tsx)**

```tsx
import { useState } from 'react'

export default () => {
    let [count, setCount] = useState(1)
    return <b onClick={() => setCount(count+=1)}>Counter {count}</b>
}
```

<div className="text-center text-2xl py-2 cursor-pointer select-none">
    <Counter />
</div>

### React Components in Markdown

### Rich Component Libraries

The Plugin component shows how we can easily utilize the rich components in the 
[shadcn](https://ui.shadcn.com/docs/components/accordion) and [radix-ui](https://www.radix-ui.com/)
component libraries:

**[Plugin.tsx](https://github.com/NetCoreTemplates/react-spa/blob/main/MyApp.Client/src/_posts/components/Plugin.tsx)**

```tsx
import { Button as PrimaryButton } from "@/components/ui/button"
import { Dialog, DialogContent, DialogDescription, DialogHeader,
  DialogTitle, DialogTrigger
} from "@/components/ui/dialog"

export default () => {
  return (<Dialog>
      <DialogTrigger asChild>
        <PrimaryButton>Open Modal</PrimaryButton>
      </DialogTrigger>
      <DialogContent>
        <DialogHeader>
          <DialogTitle>Greetings</DialogTitle>
          <DialogDescription className="p-4 text-xl">
            Hello @servicestack/vue!
          </DialogDescription>
        </DialogHeader>
      </DialogContent>
    </Dialog>)
}
```

Embeddable in `*.mdx` documents with:

```html
import Plugin from "./components/Plugin"

<Plugin />
```

<div className="text-center text-2xl py-8">
    <Plugin id="plugin" />
</div>

### End-to-end Typed API Development

For better IDE intelli-sense during development, ServiceStack's 
[TypeScript Add ServiceStack Reference](https://docs.servicestack.net/typescript-add-servicestack-reference) feature
lets you generated Typed DTOs of your .NET APIs with:

:::sh
npm run dtos
:::

That can be used in **@servicestack/client** generic typed `JsonServiceClient`
to unlock valuable static analysis type-checking and intelli-sense benefits during development:

```js
import { Hello } from '@/dtos'
client.api(new Hello({ name }))
```

You'll typically use all these libraries in your **API-enabled** components as seen in the
**HelloApi** component which calls the `Hello` API on each key press:

**[HelloApi.tsx](https://github.com/NetCoreTemplates/react-spa/blob/main/MyApp.Client/src/_posts/components/HelloApi.tsx)**

```tsx
import { useState, useEffect } from "react"
import { JsonServiceClient } from "@servicestack/client"
import { TextInput } from "@/components/Form"
import { Hello } from "@/dtos"

type Props = { value: string }
export default ({ value }:Props) => {
    const [name, setName] = useState(value)
    const [result, setResult] = useState('')
    
    const client = new JsonServiceClient()
    useEffect(() => {
        (async () => {
            let api = await client.api(new Hello({ name }))
            if (api.response) {
                setResult(api.response.result)
            }
        })()
    }, [name])

    return (<div className="my-8 max-w-fit mx-auto">
      <TextInput value={name} onChange={e=>setName(e.target.value)}/>
      <b className="my-2 block text-center text-lg">{result}</b>
    </div>)
}
```

```tsx
<HelloApi value="React" />
```
